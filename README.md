# Git Relearning Project (NPB Version)

## 概要

本リポジトリは、GitおよびGitHubの理解を深めるための「リベンジ学習プロジェクト」である。

以前、別リポジトリ（git-learning-log）において  
『いちばんやさしいGit＆GitHubの教本』に沿って以下の内容を学習した：

- Gitの基本操作  
- ファイルのバージョン管理  
- ローカルとリモートの行き来  
- ブランチを使ったファイル更新  
- 複数ブランチの操作  
- コンフリクト対応  

しかし、最終的に理解が不十分なまま一周してしまった。

---

## 本リポジトリの目的

本リポジトリは、その学習のリベンジとして作成したものである。  
教科書の内容をなぞるだけでなく、「実際に理解すること」を目的として再構築した。

---

## 前回の問題点

理解が不十分だった主な原因は以下の通り：

### ① 役割の混在

教科書では

- 複数人で役割分担する  
- もしくは1人で1役に徹する  

ことが前提となっていた。

しかし実際には、

- 1人で管理者と編集者の両方を担当  
- しかも1台のPCで操作  

という状態で進めてしまった。

その結果、

- SSH鍵認証の混乱  
- 自分が今どのリポジトリ・どの役割にいるのか不明になる  

といった問題に時間を取られ、  
本来理解すべきGitの挙動に集中できなかった。

---

### ② fork構成の誤解

もう一つの大きな問題は、fork構成に対する誤解である。

- 自分のローカルからブランチをpush  
- プルリクエストを作成  
- 管理者がマージ  

という流れの中で、

👉 マージ結果は「本体リポジトリ」にのみ反映される  

にも関わらず、

👉 自分のリモートリポジトリにも自動反映されると誤解していた  

その状態のまま学習を進めたため、

- pullやmergeを繰り返すうちに  
- 教科書の内容と挙動が噛み合わなくなる  

という状況に陥った。

---

## 学習方針の変更

この問題を踏まえ、今回は以下のように学習方法を改善した：

### ① 環境の分離

- PCを2台使用  
- 管理者と編集者を明確に分離  

→ 役割の混在を防止

---

### ② 素材の変更

前回：
- 教科書付属のリポジトリをfork

今回：
- 自分でWebページを作成  

内容：

👉 「あるプロ野球球団の1シーズン成績をまとめたWebページ」

実際のファイル例：  
:contentReference[oaicite:0]{index=0}

---

### ③ シナリオベースの学習

各チャプターごとに具体的な変更シナリオを設定：

例：
- 選手の打撃成績を追加
- 表に列を追加
- コンフリクトを意図的に発生させる

→ 実際の開発に近い形で操作を再現

---

## 結果

今回の方法では、

- ほぼ詰まることなく操作を再現できた  
- 意図した挙動を正しく発生させることができた  

結果として、

👉 GitおよびGitHubに対する理解度は大幅に向上した  

と言える。

---

## 補足

本リポジトリには以下が含まれる：

- 各チャプターのサマリー  
- 学習ログ  
- コマンドライン整理  
- 実際に操作したWebファイル  

これらを通じて、Gitの挙動を「手順」ではなく「構造」として理解することを目指している。
